# Corsair Research Engine — Specification

This is the frozen contract for the Corsair Research Engine, the live-web enrichment layer that transforms every blueprint into a continuously cited research target.

---

## 1. Mandate

For every blueprint in `phase2/data/blueprints.json`, produce and maintain a `research_envelope` JSON file at `phase2/data/research/<blueprint_id>.json` that conforms to `schemas/research.schema.json`. Envelopes are versioned; each re-crawl appends to a `history` array so signal drift is observable.

## 3. Source-Class Taxonomy (and Trust Hierarchy)

```
peer_paper > regulatory > yc_rfs ≈ a16z > major_news > founder_writeup > podcast > social
```

Canonical `source_class` enum (mirrored in `schemas/research.schema.json`):

`paper` | `post_mortem` | `news` | `regulatory` | `vc_thesis` | `yc_rfs` | `a16z` | `founder_writeup` | `dataset` | `repo` | `podcast` | `social`

Conflicts are resolved by trust order; the lower-trust source is retained but flagged with `superseded_by`.

---

## 4. Query Expansion Strategy (gbarin)

For each blueprint, derive 6–10 queries spanning these dimensions:

1. **Market size** — `"<vertical> <wedge> TAM <year>"`
2. **Named competitors** — `"<core_problem> startups <year>"`, `"<wedge> alternatives"`
3. **Recent news (90d)** — `"<wedge> funding 2026"` with date filter
4. **Academic literature** — `"<ai_leverage> <domain> arXiv"` + `"survey"`
5. **Regulatory motion** — `"<vertical> regulation <jurisdiction> <year>"`
6. **Dead-SaaS analogues** — `"<wedge> shutdown postmortem"`
7. **YC RFS / a16z overlap** — anchor-text search across both indexes
8. **Capital flow** — `"venture funding <vertical> Q1 2026"`

Queries are generated by a `query_expansion` Hermes module and recorded in the envelope's `agent_trace`.

---

## 5. Scoring Algorithm — `signal_score` (0–100)

```
signal_score =
    25 * source_diversity_norm        # number of distinct source classes / 8
  + 20 * recency_norm                  # weighted by retrieved_at, half-life 30d
  + 20 * competitor_density_norm       # active + funded competitors / 10, capped
  + 15 * regulatory_activity_norm      # filings/comment letters in last 180d
  + 10 * capital_flow_norm             # $ raised in adjacent space last 365d / $1B, capped
  + 10 * paper_velocity_norm           # arXiv abstracts in last 180d / 100, capped
```

Each component is normalized to [0,1]; final score rounded to integer.

`signal_score` is paired with a `signal_band`: `cold` (0-39), `warm` (40-69), `hot` (70-89), `nuclear` (90-100).

## 6. Budget Caps (per blueprint, per crawl)

| Resource | Cap |
| --- | --- |
| `WebSearch` calls | 20 |
| `WebFetch` calls | 30 |
| Browser navigations (`cursor-ide-browser`) | 5 |
| Apify actor runs | 5 |
| Total wall time | 5 minutes |
| Total cost (estimate) | $0.50 |

DAG halts and emits partial envelope with `truncated: true` if cap exceeded; envelope routed to HITL queue.

## 7. Freshness Policy (TTL by source class)

| `source_class` | TTL |
| --- | --- |
| `news` | 24h |
| `social` | 24h |
| `regulatory` | 7d |
| `vc_thesis` | 7d |
| `yc_rfs` | 14d |
| `a16z` | 14d |
| `founder_writeup` | 30d |

| `repo` | 7d |
| `dataset` | 30d |
| `podcast` | 30d |

App surfaces a `Stale` badge per envelope when `now - last_crawled_at > TTL_min` for any cited source class.

---

## 8. HITL Gates

The DAG halts and routes to operator approval at:

1. Overwriting an existing envelope where `signal_score` delta > 25.
2. Citing a paywalled source.
3. Citing a source flagged `low_trust` by the trust hierarchy.
4. Crawl budget exceeded (`truncated: true`).
5. >20% of citations fail validation (`needs_review` queue).

---

## 9. Cache & Politeness

- Content-addressed cache key: `sha256(url + canonical_body)`.
- Path: `phase2/data/research/_cache/<sha256>.{md,html,pdf}`.
- Honors `robots.txt`; refuses to fetch `Disallow`'d paths.
- 1 req/sec per host (configurable per host).
- User-agent: `CorsairResearchBot/1.0 (+https://idea-browser.local/about)`.
- Paywalled sources: skip by default; operator-supplied cookies allowed via `agents/research/cookies/<host>.json`.

---

## 10. Provenance Discipline

Every claim surfaced in an envelope's `summary`, `competitors[].one_liner`, or `regulatory_landscape` carries a `citation_id` pointing to a record in `citations[]`. The schema rejects envelopes containing claims without a `citation_id`. The Corsair does not bluff.

---

## 11. Determinism & Reproducibility

- DAGs are seeded by `(blueprint_id, crawl_date_utc)`.
- Re-runs against the same seed and same source state produce identical envelopes.
- Source state changes (new article, removed page) are reflected in the next crawl and recorded in the `history` diff.

---

## 12. Failure Modes (and routing)

| Mode | Routing |
| --- | --- |
| Tool timeout | retry x2, then `partial: true`, HITL |
| 4xx/5xx fetch | log + skip; record in `agent_trace.errors[]` |
| robots.txt disallow | skip + record |
| Paywall detected (heuristic) | skip + flag |
| >20% citation failures | `needs_review` queue, don't publish |
| Schema validation failure | reject envelope, hold previous version |

---

## 13. Run Modes

| Mode | When | Surface |
| --- | --- | --- |
| `seed` | Phase 2.5 initial backfill across 130 blueprints | CLI / one-shot script |
| `refresh` | Per-blueprint re-crawl on operator demand | Detail page → Deep Research tab |
| `freeform` | Free-form research from `/research` surface | Streaming agent trace |
| `scheduled` | Nightly TTL-driven refresh of stale envelopes | Cron / Inngest |

---

## 14. Output Contract

Each envelope file at `phase2/data/research/<id>.json` validates against `schemas/research.schema.json`. Required top-level keys:

`blueprint_id`, `last_crawled_at`, `signal_score`, `signal_band`, `summary`, `citations[]`, `competitors[]`, `papers[]`, `dead_saas_matches[]`, `yc_rfs_matches[]`, `a16z_thesis_matches[]`, `regulatory_landscape`, `agent_trace`, `confidence`, `history[]`, `version`.
